## 7. Usage Examples
Here are JSON schema examples demonstrating the usage of the design API:
```json
{
  "blockType": "Block Type 1",
  "variants": ["Variant A", "Variant B"]
}
```

## 8. API Response Format
The API returns responses in the following JSON format:
```json
{
  "status": "success",
  "data": { /* Response data */ }
}
```

## 9. Validation Rules
Validation ensures that all inputs to the API meet specified criteria:
- Field types must match
- Required fields cannot be empty

## 10. Best Practices for Gemini
When generating page schemas, the following best practices should be followed:
- Follow the naming conventions strictly
- Use only the defined blocks and surfaces
- Validate input data thoroughly
